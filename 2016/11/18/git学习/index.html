git学习

为什么学git

我的现状和经历
  • 上一门课或者自学一个主题，我习惯写电子笔记，还有最近归纳总结的博客文档。
  • 过去写demo修改完善它的时候总是不断复制粘帖，最后弄的很混乱 也很焦虑。
  • 我在写我博客搭建的笔记时，发现才过1个半月好多东西都忘了。
  • 引用网上别人的话”怎样让自己的心静下来？整理干净电脑（特别是桌面）中文件夹。” 




我的观点
  • 希望能以某种方式当场纪录作业的流程，事后据此留下一个说明文档，而不是以学会的立场跳步骤的记录。
  • 个人觉得工科的很多东西不需要死记硬背，因为作用范围太小，一个流程基本上只能解决特定的问题．不像数学，学会了这门形式科学的一个小领域就能解决一个方面的问题。
  • 虽然不需要死记硬背，但记录是必须的，自己花费了大量的时间去检索资料，处理问题，解决莫名其妙的坑，如果不记录，那么以后还得重头再折腾一遍。




知识与经验
  • 如果把学习知识定义为牢记教科书里的术语，掌握一学科的内在规律，能通过教科书归纳的规律演绎出特定的推论。把经验定义为将教科书的推论灵活应用于实际和工具的熟练使用以及大量的试错。
  • 知识偏向于算法和计算机专业课；经验偏向于将代码东拼西凑，使用好各种接口工具平台使代码跑起来。
  • 由于种种原因，我知识浅薄，经验又非常不足，但github提供了一个平台，可以直接学习很多人的经验和思路。




git的作用
  • 一个是创建分支以便无后顾之忧的折腾。
  • 在commit时留下准确说明，方便事后整理。
  • 每遇到一个较大的改动都可以建立分支甚至多个分支，最后只需要合并改动就行了，一种高级的偷懒方式。










怎样学的git

之前在写博客的时候只用了三条git命令来推送到Github仓库
  • git add .
  • git commit -m “update”
  • git push origin master




11月14日到12月10日是四周的企业课，花了四天抽空学习Git
  • 照着廖雪峰的Git教程操作了一遍。
  • 看了一遍Github入门与实践。
  • 再看了半本Pro Git。
  • 然后刷了两遍Learn Git Branching。
  • 最后把Pro Git看了一遍。(还是好多不懂，看了就忘)
  • 刷了Learn Git Branching 后对廖雪峰的教程理解更加清楚了，廖雪峰的教程很易懂容易记住很实用，至于其它的内容留个印象，以后遇到了估计还要Google，至于命令也不需要死记硬背，查询help就可以了，不过我觉得我还得学一下计算机专业英语，我这专业没这门课。
