Tiny6410裸机开发

开发环境

参考手册

  • 友善之臂的Tiny6410硬件手册
  • 友善之臂的使用 J-Link 在 ADS 下调试 Mini6410 裸机程序的详细步骤
  • TinyADK-1312B-Schematic(外围电路)
  • Tiny6410-1308-Schmetic(核心板电路)
  • 三星S3C6410用户手册(中文版)

硬件

  • 广州友善之臂 Tiny6410开发板
  • CPU：三星S3C6410
  • CPU内核型号：ARM11 76JFZ
  • CPU内核系列：ARM11
  • CPU内核体系结构：ARM V6
  • J-link ARM仿真器
  • usb转换器,将串口连接到开发机usb上

开发软件都部署到一个windows xp虚拟机内

  • 编码,编译在CodeWarrior for RVDS
  • AXD Debugger v1.3.1
  • dnw1.01显示串口的信息



需求分析

制作可以在屏幕上显示电子时钟系统，分为三种工作模式：

  • 正常工作模式。显示时间，掉电不丢失。
  • 时钟设置模式。在此模式下可以设置当前的时间，包括年、月、日、时、分、秒。
  • 闹铃设置模式。在此模式下设置闹铃时间，包括小时和分钟，还可以设置闹铃开关，如果关闭开关，则时间到时不提醒。
  • 如果闹铃开关打开，当设置时间到时，蜂鸣器响，通过按键可关闭蜂鸣器。
  • 通过串口实现当前时间的输出。




模块设计

GPIO:蜂鸣器

根据原理图和手册可知,先将GPFCON[29:28] = 0b00设置为输出,GPFDAT[14] = 0b01设置高电平时鸣叫。


GPIO:按键模块

  • 用查询方式实现按键驱动
  • 按键：按下，电路导通；未按下，电路断开；
  • 按键一端接地（0低电平），一端（控制端）通过上拉电阻接到高电平（1， 3.3V）
  • 按键未按下时，控制端为1，高电平；
  • 按键按下后，控制端为0，低电平
  • K1 –GPN0 输入 1（未按下） 0（按下）
  • K2-GPN1 输入
  • K3-GPN2 输入
  • K4-GPN3 输入
  • 初始化为:GPNCON[7：0] = 0b00 00 00 00
  • 读GPNDAT[3:0] = 0b1111, 变0的那位就是对应按下的按键,按键驱动核心就是读取GPNDAT&0x0F这一值。在调试过程中,通过观察内存里0x7F008834的值,发现两个键同按或者更多键同按是可以显示预期的值的,我据此在业务逻辑设计了三种工作模式。


UART串口通讯

  • 1初始化GPIO GPA为TxD，RxD
  • GPACON[7:0]=0b 0010 0010
  • 位移掩码GPACON&=~（0xf<<0）&~（0xf<<4）
  • GPACON|=（0x1<<1）|（0x1<<5）
  • 2初始化通信协议（数据帧格式（8N1），波特率115200(PCLK=66.5MHz)）

发送和接受消息就是调用老师给的接口了。


RTC实时时钟

Tiny6410外围电路上有一个比较大的纽扣电池,即使板子断电,系统时间还继续运行。只需要读取和写入对应地址的值就行,不过寄存器里的编码是8421BCD编码,平常显示的时候需要转换为十进制DEC编码。

注:RTC年份范围是0~99,再加上2000,遇到年份的转换需要事先或者事后加减2000。
